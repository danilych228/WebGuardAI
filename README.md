# #КиберПраво: твой 🛡️ в сети
# Руководство пользователя

**WebGuardAI** — это универсальный инструмент для статического анализа кода (SAST), предназначенный для выявления потенциальных уязвимостей, логических ошибок и небезопасных практик программирования. Он предоставляет два основных режима работы: консольный интерфейс (CLI) для автоматизации и графический интерфейс (GUI) для интерактивного использования.

---

## ✨ Основные возможности

*   **AI-Анализ**: Глубокое сканирование кода с использованием искусственного интеллекта для обнаружения широкого спектра уязвимостей.
*   **Сканирование локальных директорий**: Анализ проектов, расположенных на вашем компьютере.
*   **Интеграция с GitHub**: Сканирование удаленных репозиториев GitHub по их URL **без необходимости скачивания файлов** на локальный диск.
*   **Гибкие режимы работы**: Возможность выбора между командной строкой и графическим интерфейсом.
*   **Детальная отчетность**: Сохранение результатов сканирования в текстовом (`.txt`) или PDF (`.pdf`) формате.
*   **Уведомления по электронной почте**: Автоматическая отправка отчетов о сканировании через SMTP.
*   **Поддерживаемые языки**: Анализ файлов с расширениями .py, .js, .php, .html, .java, .c, .cpp, .cs, .go, .rb, .ts.
*   **Полная локализация**: Весь интерфейс и сообщения программы представлены на русском языке.

## Установка Windows/Linux

Для использования WebGuardAI выполните следующие шаги:

1.  **Получите файл `webguard.py`**:
    Этот файл содержит всю программу. Вы можете получить его из предоставленного архива.

2.  **Установите необходимые зависимости**:
    Откройте терминал Linux и выполните следующую команду для установки всех требуемых библиотек:
    ```bash
    sudo apt install git python3-pip libreoffice
    git clone https://github.com/danilych228/WebGuardAI.git
    cd WebGuardAI
    pip install -r requirements.txt
    ```
    
    Для установки на Windows требуется скачать последнюю версию Python и скачать репозиторий проекта GitHub.
Ссылка на проект: https://github.com/danilych228/WebGuardAI.git

## Использование: Руководство по командам

Запуск программы осуществляется через файл `webguard.py` с использованием интерпретатора Python. Вы можете использовать различные аргументы для выбора режима работы и настройки дополнительных опций.

### Общий синтаксис команды

```bash
python webguard.py [РЕЖИМ_РАБОТЫ] [ОПЦИИ_ОТЧЕТА] [ОПЦИИ_ПОЧТЫ]
```

### 1. Режимы работы

Вы должны выбрать **один** из следующих режимов. Эти аргументы являются взаимоисключающими.

*   **Сканирование локальной папки** (`-d` или `--dir`)
    Используйте этот аргумент для сканирования директории, расположенной на вашем компьютере. Укажите полный или относительный путь к папке.
    ```bash
    python webguard.py -d /путь/к/вашему/проекту
    ```

*   **Сканирование GitHub репозитория** (`-u` или `--url`)
    Используйте этот аргумент для анализа удаленного репозитория GitHub. Укажите полный URL репозитория. Программа получит список файлов и их содержимое напрямую через GitHub API, не сохраняя их на ваш диск.
    ```bash
    python webguard.py -u https://github.com/владелец/репозиторий
    ```

*   **Запуск графического интерфейса (GUI)** (`--gui`)
    Используйте этот аргумент для запуска приложения в режиме графического интерфейса. Все функции сканирования будут доступны через удобный визуальный интерфейс.
    ```bash
    python webguard.py --gui
    ```

### 2. Опции отчета

Эти опции можно комбинировать с режимами сканирования (`-d` или `-u`). Они позволяют настроить сохранение результатов анализа.

*   **Сохранить отчет в PDF** (`--pdf`)
    Сохраняет детальный отчет о сканировании в формате PDF. Вы можете указать имя файла для отчета. Если имя не указано, будет использовано имя по умолчанию `report.pdf`.
    ```bash
    python webguard.py -d /путь/к/проекту --pdf мой_отчет.pdf
    ```

*   **Сохранить отчет в TXT** (`--txt`)
    Сохраняет отчет в простом текстовом формате. По умолчанию, текстовый отчет всегда генерируется как `report.txt`. Вы можете указать другое имя файла, если это необходимо.
    ```bash
    python webguard.py -u https://github.com/владелец/репозиторий --txt результаты_гитхаб.txt
    ```

### 3. Опции почты

Эти опции используются для автоматической отправки отчета о сканировании по электронной почте. Они также комбинируются с режимами сканирования (`-d` или `-u`).

*   **Адрес получателя** (`--email`)
    **Обязательный** аргумент при использовании функций отправки почты. Указывает адрес электронной почты, на который будет отправлен отчет.
    ```bash
    python webguard.py -d /путь/к/проекту --email получатель@example.com
    ```

*   **SMTP сервер** (`--smtp-server`)
    Указывает адрес SMTP-сервера для отправки почты. По умолчанию используется `smtp.gmail.com`.
    ```bash
    python webguard.py -u https://github.com/владелец/репозиторий --email получатель@example.com --smtp-server smtp.mail.ru
    ```

*   **Порт SMTP** (`--smtp-port`)
    Указывает порт SMTP-сервера. По умолчанию используется порт `587` (для защищенного соединения TLS).
    ```bash
    python webguard.py -d /путь/к/проекту --email получатель@example.com --smtp-port 465
    ```

*   **Email отправителя** (`--sender-email`)
    Адрес электронной почты, с которого будет отправлен отчет. **Этот аргумент опционален**. Если он не указан, программа попытается отправить письмо без аутентификации. Это может быть полезно для SMTP-серверов, которые не требуют авторизации (например, некоторые корпоративные или локальные реле).
    ```bash
    python webguard.py -u https://github.com/владелец/репозиторий --email получатель@example.com --sender-email мой_адрес@gmail.com
    ```

*   **Пароль отправителя** (`--sender-password`)
    Пароль для учетной записи отправителя. **Этот аргумент опционален**. Он требуется, если вы указали `--sender-email` и ваш SMTP-сервер требует аутентификации. Для сервисов вроде Gmail часто необходимо использовать [пароли приложений](https://support.google.com/accounts/answer/185833?hl=ru) вместо основного пароля.
    ```bash
    python webguard.py -d /путь/к/проекту --email получатель@example.com --sender-email мой_адрес@gmail.com --sender-password "ваш_пароль_приложения"
    ```

### 4. Комбинирование опций

Вы можете комбинировать различные опции отчета и почты с выбранным режимом сканирования для выполнения комплексных задач.

*   **Пример: Сканирование локальной папки, сохранение в PDF и отправка по почте**
    ```bash
    python webguard.py -d /путь/к/проекту --pdf отчет_сканирования.pdf --email администратор@example.com --smtp-server smtp.mycompany.com --smtp-port 587 --sender-email мой_корп_адрес@mycompany.com --sender-password "мой_пароль"
    ```

*   **Пример: Сканирование GitHub репозитория, сохранение в TXT и отправка по почте без аутентификации (если SMTP-сервер это позволяет)**
    ```bash
    python webguard.py -u https://github.com/другой/репозиторий --txt github_результаты.txt --email команда_безопасности@example.com --smtp-server локальный_smtp.com --smtp-port 25
    ```
